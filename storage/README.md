# Storage (Filesystem)

Flat filesystem storage for device images and video archives. No cloud storage, no organized directory structure, no retention policies.

## Layout

```
storage/
├── images/     ← Uploaded device images (UUID.jpg files)
├── video/      ← Generated video archives (UUID.mp4 files)
└── storage.txt ← Empty file
```

## File Naming

- **Images**: `uuid.v4() + .jpg` — flat, no subdirectories, no user/device organization
- **Videos**: `uuid.v4() + .mp4` — flat, no subdirectories

## How Files Are Written

### Image Upload (Device API)

`hs_deviceAPI.js` uses `multer.diskStorage`:

```js
var cameraImageStorage = multer.diskStorage({
  destination: function (req, file, cb) {
    cb(null, '/projects/homeSecurity/server/storage/images');
  },
  filename: function (req, file, cb) {
    cb(null, uuid.v4() + path.extname(file.originalname));
  },
});
```

The manager also has a `storeFile()` function that writes image buffers directly via `fs.writeFile()`.

### Video Generation (Service + Manager)

Videos are generated by `videoshow`/ffmpeg and saved to the video directory by the video manager and archive scheduler.

## Path Constants (in `ioc.manager.js`)

| Constant | Path |
|----------|------|
| `CAMERA_IMAGE_STORAGE_PATH` | `/projects/homeSecurity/server/storage/images` |
| `CAMERA_VIDEO_STORAGE_PATH` | `/projects/homeSecurity/server/storage/video` |

## Storage Usage Tracking

The `storage.manager.js` (in `manager/`) calculates per-user storage usage using `fast-folder-size` on the user's image/video subdirectories. The cron job in `service/service.js` runs this periodically and updates `storageUsedSize`/`storageRemainedSize` on the User document.

## Notable

- All paths are hardcoded to `/projects/homeSecurity/server/storage/` — not configurable.
- No S3, no signed URLs, no CDN upload — images are referenced via a separate CDN hostname (`cdn.hushiar.com`) in the admin HTML output, but files are stored locally.
- No directory organization by user, device, or date — everything is flat.
- No cleanup/retention policy — files accumulate indefinitely.
- `storage.txt` is an empty file with no known purpose.
